## 关于自我

### "笨"是深度处理器的副作用
- 不是反应慢，是在建立更底层的认知架构
- 在复杂问题上，深度会超越所有"聪明"的人
- 停止用"反应速度"评价自己，改用"问题理解深度"

### "半途而废"是高效的目标筛选
- 不是不能坚持，是拒绝在无意义的事情上坚持
- 当价值连接清晰时（如 Claude Code），能爆发惊人毅力
- 启动项目前回答: "做成了，谁会因此受益？"

### 嫉妒是路标
- 嫉妒"不上班"= 渴望自主性
- 嫉妒"有钱"= 渴望选择权
- 不是可耻的情绪，是指向真正想去的地方

### 情绪消费是信号
- 冲动消费 = 内心空虚/焦虑的外在表现
- 用"创造"替代"购买"来填补情感空缺
- 尝鲜预算应视为"认知投资"，要有产出

## 关于工作

### 技术自嗨陷阱
- 纯技术探索如果不连接用户需求，动力会断
- 避免"为了技术而技术"
- 始终问: 这个技术解决了谁的什么问题？

### Build in Public 是最佳策略
- 完美匹配"建造者-表演者"混合天赋
- 创造 -> 展示 -> 反馈 -> 再创造的闭环
- 不需要完美，需要持续

### spawn/PTY 包装 CLI 是死路
- 不稳定、进程管理复杂、延迟高
- 直接调用 API 重写也有问题（丢失核心竞争力）
- 最佳路径: 反编译 + 模块提取

## 关于财务

### 收入 - 支出 = 几乎为 0 是核心问题
- 不是收入不够，是情绪消费吞噬了所有增量
- 控制情绪消费比增加收入更紧急
- 被动收入 > 打工收入（长期）

### 消费不是问题，没有复利才是
- 花钱本身不坏，关键是花钱的方式有没有产生复利
- 认知投资（尝试新工具）是好的消费
- 纯情绪消费（冲动购买）是需要控制的

## 关于人际

### 多元认可账户
- 不把所有认可需求放在一个人/一个地方
- 技术社区 + 内容粉丝 + 朋友 + 自我认可
- 自我认可是终极目标，但要慢慢来
